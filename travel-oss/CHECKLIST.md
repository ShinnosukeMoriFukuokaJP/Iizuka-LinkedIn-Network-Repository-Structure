# Travel OSS Checklist

旅行プロジェクトを OSS として完成させるためのチェックリスト。
完了したら `[x]` に変更して GitHub に push してください。

---

> **個人情報・プライバシーに関する重要事項**
>
> 旅行記録をGitHubで公開する際は以下に注意してください。
>
> - **他者の個人名**：氏名・連絡先など個人を特定できる情報は、本人の同意なくGitHub公開リポジトリに記載しないでください（個人情報保護法）。
> - **写真・画像**：第三者の顔が映り込んだ写真の無断公開は、プライバシー権・肖像権の侵害となる可能性があります。モザイク処理または掲載自体を避けてください。
> - **店舗・施設の情報**：事実に基づく記録は問題ありませんが、虚偽の内容や名誉を傷つける内容は名誉毀損に該当する可能性があります。

---

## Phase 1｜旅行前（設計）

### リポジトリ設計
- [ ] リポジトリ名を決定（例：`kyoto-oss-2025`）
- [ ] GitHub で新規リポジトリ作成（Public）
- [ ] `templates/repo-readme.md` をカスタマイズ
- [ ] README を push
- [ ] GitHub Pages を有効化（Settings → Pages → main branch）

### 記録準備
- [ ] `templates/travel-record.md` を iPhone メモにコピー
- [ ] 記録したい項目を事前に決める（食事・交通・場所・気づき）
- [ ] Claude システムプロンプトを保存（`SYSTEM_PROMPT.md` 参照）
- [ ] **公開範囲を決める**（個人名・写真等の取り扱い方針を事前確認）

### JLI-OS 連携（任意）
- [ ] 旅行先が JLI-OS の対象都市か確認
- [ ] `templates/city-node.md` を準備

---

## Phase 2｜旅行中（記録）

### 毎日の記録
- [ ] 朝：その日の予定を Markdown に書き出す
- [ ] 移動中：気づきをメモ（箇条書きで OK）
- [ ] 夜：Claude で整形 → GitHub に push

### コンテンツ収集
- [ ] 地名・店名は正式表記で記録する
- [ ] 数字（価格・距離・時間）は具体的に記録する
- [ ] **他者の氏名は記録しない**（本人同意がある場合を除く）
- [ ] **写真は第三者が特定できないものを選ぶ**（風景・料理・建物等）
- [ ] 「他人が再現できるか」を意識して書く

### GitHub 管理
- [ ] 1日1回以上 push する習慣をつける
- [ ] コミットメッセージは日付 + 内容（例：`2026-06-27: 京都Day1記録追加`）

---

## Phase 3｜旅行後（公開）

### 公開前の個人情報チェック
- [ ] ファイル内に他者の個人名・連絡先が含まれていないか確認
- [ ] 第三者の顔が映った写真が含まれていないか確認
- [ ] 事実と異なる内容・誤解を招く内容がないか確認

### ドキュメント完成
- [ ] `travel-record.md` を最終整形
- [ ] README を公開品質に仕上げる
- [ ] `templates/city-node.md` を完成させる（JLI-OS用）

### 公開
- [ ] GitHub Pages の表示確認
- [ ] リポジトリに Topics を設定（例：`travel`, `oss`, `japan`, `markdown`）
- [ ] LICENSE ファイル追加（MIT推奨）
- [ ] Social Preview 画像設定（任意）

### 発信
- [ ] `templates/linkedin-post.md` を使って LinkedIn 投稿（英語）
- [ ] 日本語版も投稿
- [ ] リポジトリ URL を投稿に含める

### 次の作品へ
- [ ] `SYSTEM_PROMPT.md` の「⑩次に作るべき作品」から次を選ぶ
- [ ] Version 2 の改善案をリポジトリの Issues に登録

---

## 完成定義

以下がすべて揃った状態を「完成」とする：

| 成果物 | 状態 |
|--------|------|
| GitHub Repository | Public で公開済み |
| README | そのまま読める品質 |
| GitHub Pages | URL でアクセス可能 |
| Markdown記事 | 旅行記録が整形済み |
| LinkedIn投稿 | 英語・日本語で発信済み |
| テンプレート | 他人がForkして使える状態 |
