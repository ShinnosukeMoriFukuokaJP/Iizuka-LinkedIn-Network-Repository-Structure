# LinkedIn統合システム

## 問題

LinkedInプロフィールとGitHubリポジトリは**別々の、リンクされていない成果物**として存在している。  
一方を見つけた採用担当者は、もう一方を簡単に理解できない。  
LinkedInの表面層からは技術的な深さが見えない。

## 解決策

以下の**双方向マッピング**を構築：
- LinkedInナレーティブ（About、Featured、投稿）
- GitHub実装（プロジェクト、README、アーキテクチャ文書）

LinkedInの各要素はGitHubの証明を指す。  
各GitHubプロジェクトはLinkedInのストーリーに紐づく。

## インパクト

- 「この人は実際に何を作っているのか？」という曖昧さを排除
- 評価時間を数分から数十秒に短縮
- **自己強化する信頼ループ**を構築

## 設計原則

1. **スキル羅列禁止** — 成果とシステムのみ
2. **すべての主張にGitHubリンク** — 未検証の主張なし
3. **ストーリー > 履歴書** — ナレーティブ構造が箇条書き積み上げに勝る
4. **循環、線形ではない** — LinkedInとGitHubが互いを強化する
