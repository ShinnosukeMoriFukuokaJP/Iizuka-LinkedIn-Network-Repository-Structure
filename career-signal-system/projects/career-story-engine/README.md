# Career Story Engine

## Problem

Technical work does not automatically translate into **evaluable value**.  
Engineers often describe *what* they built, not *why it matters*.  
This creates a gap between execution quality and perceived impact.

## Solution

A structured conversion framework:

```
Technical Output → Narrative Frame → Career Signal
```

Every project goes through three filters:

1. **Problem filter** — What real friction did this solve?
2. **Solution filter** — What did you actually build or design?
3. **Impact filter** — What changed as a result?

## Impact

- Engineers with identical skills but different narrative structures
  are evaluated **very differently** by recruiters
- This engine closes that gap with a repeatable, honest framework
- No credential inflation — only structural clarity

## Core Logic

### PSI Transform

| Layer | Question | Output |
|-------|----------|--------|
| Problem | What was broken or missing? | 1-sentence problem statement |
| Solution | What did you build? | System description (no jargon) |
| Impact | What changed? | Outcome statement (metric or structural) |

## Usage Pattern

```
1. Complete any project (even small ones)
2. Run it through PSI Transform
3. Commit README to GitHub
4. Write LinkedIn post with GitHub link
5. Publish → observe engagement
6. Add case study if project resonates
7. Repeat
```

## Principle

> The goal is not to make small projects look big.  
> The goal is to make the *value* of real work *visible*.
