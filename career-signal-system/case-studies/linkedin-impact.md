# Case Study: LinkedIn Profile Impact Design

## Context

Iizuka City, Fukuoka Prefecture — regional professional ecosystem
where LinkedIn penetration is lower than Tokyo, creating both
a challenge (fewer connections) and an opportunity (less noise).

## Problem

A LinkedIn profile without GitHub integration appears as:
- A list of job titles
- A list of skills (unverified)
- No proof of execution

## Solution Implemented

1. **Headline redesign** — system builder identity + geographic anchor
2. **About section** — story-first, outcome-anchored, GitHub link embedded
3. **Featured section** — GitHub repo as primary featured item
4. **Post strategy** — system updates as LinkedIn content

## Impact

| Metric | Before | After |
|--------|--------|-------|
| Profile clarity | Generic | System-builder identity |
| GitHub visibility | Zero | Direct from profile |
| Regional positioning | Invisible | Fukuoka/Kyushu anchor |

## Key Learning

> The profile is not a resume.  
> The profile is a **landing page**.
