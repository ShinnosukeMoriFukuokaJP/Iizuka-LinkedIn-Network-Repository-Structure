# Case Study: System Design as Career Signal

## The Core Insight

Two developers with identical skills will be evaluated differently
based on how they *document and present* their work.

This is not about deception. This is about **signal clarity**.

## What Recruiters Evaluate in 30 Seconds

1. Can I understand what this person builds?
2. Does their GitHub reflect their LinkedIn?
3. Is there evidence of structured thinking?
4. Would this person be easy to work with?

None of these questions are answered by a list of technologies.

## Architecture Decision: Why GitHub + LinkedIn

| Option | Pro | Con |
|--------|-----|-----|
| Portfolio website | Looks polished | Costs money, requires maintenance |
| LinkedIn only | Easy to update | No proof layer |
| GitHub only | Full technical proof | No narrative layer |
| **GitHub + LinkedIn** | **Proof + narrative** | **Requires consistent updates** |

The combination wins because it is **self-verifying** —
LinkedIn makes a claim, GitHub proves it.

## Reproducibility

This framework can be cloned and adapted:

1. Fork this repository
2. Replace project content with your own
3. Apply PSI Transform to your projects
4. Link LinkedIn profile to this repo
5. Start the posting cycle

> The framework is the product. The career signal is the output.
